# AGENTS.md

Project: **notagrafo** (`NOTA`)

This Mnema project is managed by the `@felipesauer/mnema` MCP server. Anything an agent does in this repository must flow through the tools exposed by the server.

## Operating principles

1. Start every session with the `context_bootstrap` tool.
2. Wrap every batch of mutations in `agent_run_start` / `agent_run_end`.
3. Prefer transition tools (e.g. `task_submit`) over editing fields directly.
4. Read first, write second. The audit log records every mutation, including yours.

## Workflow: `default`

Default 7-state workflow: DRAFT → READY → IN_PROGRESS → IN_REVIEW
→ DONE, with BLOCKED and CANCELED branches. Submitting a draft
requires title, description, acceptance criteria and an estimate.
Code review is mandatory before DONE.

## Dual identity

Every mutation captures three identifiers: `actor` (the human responsible), `via` (the agent that did the work, sourced from MCP `agent_handle`), and `run` (the run id from `agent_run_start`). Mutations without an active run are rejected with `NO_ACTIVE_RUN`.

## Memory & decisions

Source of truth for agent-recorded context is SQLite: `memories_inventory` (write via `memory_record`) and `recent_observations` (write via `observation_record`). The `memory_index` / `decisions_index` fields in `context_bootstrap` are optional human-curated supplements regenerated by `mnema memory consolidate` — read for context, never write to them.

When a free-form `note_add` matures into an architectural decision, use `decision_promote_from_note(note_id, title, decision, …)` to record the ADR with a `decision_promoted_from_note` audit event linking back to the original note. Use `decision_record` directly when the decision was not preceded by a note.

## Useful CLI commands for the human

```
mnema task list                       # tasks in any state
mnema history --since=today           # what happened today
mnema watch                           # live tail of mutations
mnema inbox                           # things waiting on me
mnema agent inspect <run_id>          # detail of an agent run
```
