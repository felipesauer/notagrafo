NOTA-28 (início Sprint 7): Dockerfiles multi-stage api/worker/dashboard + nginx.conf + .dockerignore. Adaptações ao @notagrafo/* (ADR-2). ACHADOS no build real (Docker local): (1) node:20-alpine não tem build tools — libxmljs2 (nativo, api+worker) falha no node-gyp; corrigido com apk add python3 make g++ no stage deps. (2) o stage production (FROM node:20-alpine novo) tentava pnpm install --prod e RECOMPILAVA libxmljs2; resolvido copiando o workspace já construído com COPY --from=build /app ./ (node_modules com binário compilado + dist + schemas) — mais simples e rápido que reinstalar. (3) dashboard NÃO depende de core/worker; o filter ...(reticências) puxava o workspace todo e disparava o build do libxmljs2; corrigido com --filter @notagrafo/dashboard --ignore-scripts (dashboard não tem deps nativas). XSDs entram na imagem da API porque o build do core copia src/schemas→dist (NOTA-14) e o production copia tudo. As 3 imagens buildam exit 0; verificado dentro: api tem dist/schemas/xsd/v4.00 (5 XSDs) + server.js, worker tem worker.js, dashboard tem nginx try_files + index.html. Aviso de vulnerabilidades em node:20-alpine é da imagem base fixada pelo plano (não alterar versões — instrução do .plan).
