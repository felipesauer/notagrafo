Gotcha reconfirmado na Fase 1 (redesign): o Playwright roda contra baseURL http://localhost:8080 (container nginx = BUILD DE PRODUÇÃO), NÃO contra o Vite dev da 5173. Screenshots de verificação que tirei apontavam para 5173 (com HMR, código novo), mas os e2e falharam porque a 8080 servia o bundle antigo. Fix: `pnpm --filter @notagrafo/dashboard build` + `docker compose --profile app build --no-cache dashboard` + `up -d --force-recreate dashboard`, e conferir `docker inspect notagrafo-dashboard-1 --format '{{.Created}}'` (deve ser agora) e o hash do bundle em `curl -s localhost:8080 | grep index-*.js` (deve mudar) ANTES de rerodar os e2e. Depois disso os 13 specs passaram com --workers=2. Casa com a memória docker/screenshot existente. Também: `gh pr edit --title` está falhando silenciosamente neste repo por um warning de 'Projects classic deprecation' do GitHub — o título do PR #53 ficou como o da Fase 0; não-bloqueante.
