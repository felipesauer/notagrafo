NOTA-23 (Sprint 6): layout global + login do dashboard. Componentes: Sidebar (7 itens + footer usuário/logout), Header (breadcrumb via useRouterState+TITULOS, toggle tema/idioma, sino que acende com export ready), ExportBanner (polling 10s GET /export/:id, persiste via useExportStore, botão dispensar), AppLayout (compõe sidebar+header+banner+Outlet). Router reestruturado: protectedLayout é um id-route (pathless) com beforeLoad que exige isAuthenticated senão redirect to /login search {redirect: location.pathname}; as 7 rotas são filhas do layout (5 ainda Placeholder, Overview real); login com validateSearch redirect. ACHADO: com TanStack Router code-based (sem o plugin de geração de tipos), o Link só conhece os paths registrados no momento da compilação do componente — Link to='/nf' na Sidebar dava TS2322 (só aceitava '/'|'/login'); contornado com to={item.to as string}. Quando a NOTA-24+ adicionar páginas reais, considerar o plugin @tanstack/router-plugin para tipos de rota completos. vite build 129 módulos; app responde HTTP 200 em /login; test:unit 36/36, typecheck/lint/build raiz verdes. Ícones: emojis no lugar de ti-* (Tabler) para não adicionar dep — AC pede os 7 itens, não a lib.
