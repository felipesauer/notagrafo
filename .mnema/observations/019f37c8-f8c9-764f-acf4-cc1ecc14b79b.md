Achados da revisão detalhada resolvidos, commit 55b5075 no PR #53:
1. BUSCA client-side em Empresas e Produtos (antes só Notas): estado buscaLocal no Explorer.tsx (separado do q server-side de notas), input no header quando entity é empresas/produtos, passado como prop `busca`. ExplorerEmpresas filtra por razaoSocial/dígitos-CNPJ; ExplorerProdutos por descrição/NCM. Reseta em trocar(); empty state explorer.semResultados. Nota: ExplorerProdutos JÁ tinha peek com histórico de preço (usePriceHistory) + empresas ligadas (useProductCompanies) do polish anterior.
2. GRAFO legenda dinâmica (Graph.tsx Legenda): recebe {produtos,notas} e lista só tipos ativos (empresa sempre). Antes hardcoded 3 tipos → prometia 'Notas' com includeNotas OFF (default false em graph.store).
3. GRAFO rótulos de aresta sob demanda: WeightedEdge só renderiza o label quando data.focado (era `!dim`, mostrava sempre → amontoado no centro). Novo flag EdgeData.focado setado no useEffect de hover (Graph.tsx ~172): focado = há hover ativo E aresta na vizinhança. Miolo limpo, label no hover do nó.

Achados AINDA pendentes da revisão (não críticos): Empresas com poucas linhas não usa todas as stats da API no corpo da tabela (recebidas/1ª-última emissão só no peek); Produtos mostra 'Notebook 15 polegadas' repetido sem coluna diferenciadora (idUnico/emitente). 13/13 e2e verdes.
