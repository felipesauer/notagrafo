Ao rodar a suíte e2e completa contra o stack docker (profile app+demo, seed 40 NFs + usuário demo criado no seed), os 7 specs NOVOS/ampliados (overview, empresas, produtos, nf-detail, upload, nf-list×2) passaram 7/7. Mas 2 specs PRÉ-EXISTENTES (NOTA-27) falharam, revelando bugs reais: (1) GRAFO — ao navegar de Empresas/NFDetail para /grafo via Link search={{cnpj}}, o TanStack Router serializa o search como JSON e a URL fica cnpj=%2214200166000187%22 (COM ASPAS), então o grafo não encontra a empresa e o React Flow não renderiza. Correção provável: passar o cnpj como string crua na rota ou ajustar o parse do search param. (2) AUTH — login com senha errada não exibiu role=alert no timeout; o Login.tsx TEM role=alert (linha 52), então investigar se o backend retorna 401 para o usuário demo com senha errada ou se há timing. Ambos fora do escopo da NOTA-45 (que era só adicionar specs); candidatos a tasks de correção. Nota: o seed passou a criar o usuário demo (demo@notagrafo.local/demo1234) via MERGE Cypher, necessário para o login dos e2e.
