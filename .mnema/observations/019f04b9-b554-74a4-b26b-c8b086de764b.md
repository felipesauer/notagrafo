NOTA-11: nf.repository.ts com mergeNotaFiscal(driver, payload) — padrão MERGE da seção 4. @notagrafo/graph agora depende de @notagrafo/core (workspace:*); o teste de integração reusa parseNFe do core para montar o payload da fixture. Decisões/desvios do Cypher literal do plano para garantir o AC4 (reprocessar não duplica): (1) RawData usa MERGE (nf)-[:TEM_RAW]->(raw) + SET em vez de CREATE, senão reprocessar criaria múltiplos RawData; (2) CONTÉM usa MERGE por (nf)-[:CONTÉM {numeroItem}]->(prod) + SET, em vez de CREATE, para não duplicar a aresta. Helper limpar() remove undefined/null (Neo4j não aceita prop null) — datas convertidas para ISO string. Evento continua CREATE (auditoria: múltiplos eventos por NF são esperados, mas idempotência conta nós com constraint). limpar() tipado como (obj: object) porque as interfaces do core não têm index signature.
