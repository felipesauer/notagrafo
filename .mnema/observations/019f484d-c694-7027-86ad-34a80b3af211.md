NOTA-207/auditoria: o achado do explorador estático 'catch{} em alert.repository.ts:84,179 mascara erro de query Neo4j' é FALSO-POSITIVO. Ambos os catch{} envolvem apenas JSON.parse de um campo string (refs/config) com fallback sensato — a query Neo4j roda FORA do try/catch. Não há erro de Neo4j sendo engolido. Nenhuma mudança feita ali. Lição: 'catch vazio' em leitura estática precisa checar o que está DENTRO do try antes de reportar. Também sobre o README Mailpit: o .env local do autor remapeia MAILPIT_UI_PORT=8035/SMTP=1035 (como o REDIS_PORT=6390), então o container roda em 8035; o README listava 8035 na coluna host refletindo esse ambiente, mas divergia do default 8025 do .env.example. O fix alinhou o README ao DEFAULT documentado (8025/1025).
