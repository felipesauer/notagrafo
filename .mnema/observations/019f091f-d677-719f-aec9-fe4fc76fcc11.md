NOTA-29 (Sprint 7): docker-compose.yml com profiles app/demo, validado de verdade (Docker local). name: notagrafo (ADR-2). Stack completo subiu: redis/neo4j/minio healthy, api healthy, worker x2 (replicas 2), dashboard em :8080 (HTTP 200, title notagrafo), proxy nginx /api+/health funcionando. ACHADOS no ambiente do Felipe: (1) portas 6379, 6380, 1025 já ocupadas por serviços nativos do host (Redis, Mailpit) — tornei a porta de host do redis configurável via ${REDIS_PORT:-6379} (porta interna segue 6379) e subi com REDIS_PORT=16379; mailpit não é dependência do app, subi sem ele. (2) BUG do healthcheck da api: o plano usa http://localhost:3000/health, mas no Alpine/musl localhost resolve ::1 (IPv6) e a API escuta IPv4 0.0.0.0 → connection refused → api ficava unhealthy mesmo respondendo 200; corrigido para http://127.0.0.1:3000/health. (3) GAP que afeta o demo: o bucket MinIO 'notagrafo' não é criado automaticamente, então S3XmlStorage.save falha no seed (as NFes gravam no grafo mas falham no storage — runSeed conta como falha mas engole o erro). É da NOTA-13 (storage deveria garantir/criar o bucket) ou precisa de um init de bucket no compose — NÃO bloqueia a NOTA-29. (4) docker compose --profile demo run seed sozinho falha ('undefined service api') porque seed depends_on api (profile app) — uso correto é --profile app --profile demo, como o AC4 diz. .env precisa de hosts internos da rede (neo4j:7687, redis:6379, minio:9000) e está no gitignore. Criar .env a partir do .env.example.
