NOTA-30 (Sprint 7): .github/workflows/ci.yml (seção 5 do plano, adaptado a @notagrafo/*) + 2 issue templates (bug_report.yml, feature_request.yml, issue forms). CI: lint→[test-unit,test-integration]→[test-e2e,build]. Ajustes vs plano: (1) test-unit e test-integration rodam pnpm --filter @notagrafo/core build (e graph/worker no integration) ANTES dos testes, porque o validador precisa dos XSDs copiados para dist (NOTA-14) e os pacotes precisam estar buildados; (2) build job sem push (push:false), cache gha, só na main. RISCO registrado: o test-e2e sobe docker compose --profile app com DEMO=true; mas o bucket MinIO não é criado automaticamente (gap NOTA-13), então o seed grava no grafo mas falha no storage — os specs e2e foram escritos tolerantes (ex.: upload aceita 'já processada'), mas se o CI exigir seed 100% verde pode falhar. Resolver o bucket (NOTA-13) torna o e2e do CI robusto. YAML dos 3 arquivos validado (python yaml.safe_load); jobs/needs conferidos. Não dá para rodar o CI localmente (é GitHub Actions), mas os comandos individuais (lint/typecheck/test:unit/test:integration/docker build) já foram validados nas tasks anteriores.
