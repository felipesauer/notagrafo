A 2ª auditoria mediu pnpm test:coverage: lines 39.83%, functions 35.89%, branches 34.32% — MUITO abaixo do threshold de 80/80/70 configurado em vitest.config.ts. Causa (dupla-checada): a config exclui '**/*.integration.test.ts' do include de cobertura, e a maior parte do código (auth.routes 0%, nf.routes 3.57%, nf.repository 0%, queries 30-60%, queue/storage ~10%, seed/index 2.7%, stats.routes 1.75%) é exercida SÓ por testes de integração (Testcontainers). São 8 arquivos unit vs 11 de integração. Logo o threshold de 80% nunca foi exercido de verdade pela suite unit — o gate de cobertura é fantasma. Baseline funcional: unit 37/37 e integração 66/66 verdes. Para atingir 90% unit global é preciso escrever testes unitários novos (mockando driver/queue/storage) para auth, nf.routes, repository, queries, queue e seed.
