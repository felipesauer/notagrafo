DOGFOODING mnema 0.13 — fricção do processo de upgrade entre versões (relato do próprio autor): não existe UM comando que resolva todos os gaps de ambiente de uma versão para outra. Para ir de 0.12.0-alpha.0 a 0.13.0-alpha.0 foi preciso executar manualmente, em sequência descoberta por tentativa: mnema upgrade + mnema sync + mnema migrate + mnema doctor --rebuild-mirrors + mnema adopt --all (5 comandos). Cada um resolve um pedaço (versão no config / mirrors / schema do banco / reconstrução / componentes novos), nenhum orquestra o todo, e a ORDEM importa — a geração do AGENTS.md antes do memory/INDEX.md existir (achado do import 'skipped — file not found') e o rebuild-mirrors interagindo com o estado pré-migração (janela do incidente de truncation/regressão) são plausivelmente consequências de ordem livre. Sugestão: fazer de 'mnema upgrade' um orquestrador idempotente — bump → migrate → sync/rebuild → adopt (prompt p/ componentes novos da versão) → agents sync/AGENTS.md → doctor final — com --dry-run mostrando o plano e cada passo rodando só se necessário. Num alpha com release quase semanal, essa fricção multiplica a cada usuário × cada versão.
