Gotcha recorrente nas tasks de página do NOTA-EPIC-13: o tsconfig do dashboard NÃO tem noUnusedLocals, então imports não usados passam no `tsc --noEmit` (build) mas o ESLint (@typescript-eslint/no-unused-vars) barra com exit 1. O Dockerfile só roda vite build (não lint), então a imagem builda mesmo com import órfão, mas o `pnpm lint` do CI falha. LIÇÃO: rodar `pnpm lint` ANTES de `pnpm build` + `docker compose up --build` + e2e — senão o rebuild de imagem (~min) é desperdiçado quando o lint falha depois. Ordem correta por página: lint → build → docker build → e2e.
